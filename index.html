Automation test

El testing de software se ha convertido en una parte cada vez más crítica del ciclo de desarrollo de software. Tradicionalmente esta tarea se les ha confiado a grandes equipos que ejecutan test cases de forma manual. El ciclo de vida de desarrollo también es una de las partes del proceso que depende de acciones que se repitan continuamente todo el tiempo. En tiempos recientes, los equipos encontraron la clave para facilitar un proceso de deployment más rápido: automation.

La industria comenzó a ser más inclusiva con automation por el simple hecho de que aumenta la velocidad, eficiencia y cobertura sobre la aplicación en desarrollo.

El Testing automation es un conjunto de herramientas, scripts y software para ejecutar test cases predefinidos en diferentes capas y niveles de la aplicación. Esto evita que el responsable del proceso tenga que repetir las mismas acciones una y otra vez, derivando en pérdida de tiempo, que empeora en cada nueva iteración del ciclo de desarrollo cuando se agregan nuevas funcionalidades a la app. industria comenzó a ser más inclusiva con automation por el simple hecho de que aumenta la velocidad, eficiencia y cobertura sobre la aplicación en desarrollo.

El testing exploratorio y manual son pasos cruciales para productos nuevos y para configurar la eventual automatización. Sin embargo, a medida que la estrategia de desarrollo adopte marcos de desarrollo más ágiles, la estrategia de testing eventualmente tendrá que escalar para estar a la altura.
